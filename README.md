# QNLOO-SPL 主权公开许可范式官方基准仓库 / QNLOO-SPL Sovereign Public License Paradigm Official Reference Repository

本仓库是 **QNLOO-SPL（千年鹿主权公开许可范式）** 的唯一官方标准仓库，面向全球开发者和企业提供一套独立、可复用、行业通用的数字资产许可标准。

This repository is the sole official standard repository for the **QNLOO-SPL (QNLOO Sovereign Public License Paradigm)**, providing a standalone, reusable, and industry-universal digital asset licensing standard for developers and enterprises worldwide.

## 1. 范式定位 / Paradigm Positioning

QNLOO-SPL 不属于传统免费开源协议。它是一套**源码开放的主权商事价值分配范式**，完整解决数字资产确权、创作者分润、全球低成本维权、主权数字货币合规清算等全链路问题。

QNLOO-SPL is not a traditional free open-source license. It is a **sovereign commercial value distribution framework with open source code**, comprehensively addressing digital asset rights confirmation, creator revenue sharing, low-cost global enforcement, and sovereign digital currency compliance settlement.

For patent grant addendum, see the PATENTS file.

The QNLOO-SPL paradigm possesses the following five defining features, establishing clear boundaries from all existing license agreements worldwide:

| 特征 / Feature | 说明 / Description |

| **阶梯式营收分层授权体系**<br>**Progressive Revenue-Based Tiered Authorization** | 个人/学术永久免费；小微营收阈值内全豁免；合规主体超额营收固定比例分润；游离生态主体全额营收计费；大厂产品线穿透核算<br>Permanent free use for individuals and academia; full exemption for small-scale entities within revenue thresholds; fixed-percentage revenue sharing for compliant entities with excess revenue; full-revenue billing for off-chain entities; and enterprise product-line revenue penetration |
| **主权法定数字货币结算机制**<br>**Sovereign Statutory CBDC Settlement Mechanism** | 以发行主体所属国家法定数字货币为全球基准结算通道，配套过渡期安排，绑定本土企业完税与资金追溯合规义务<br>The statutory digital currency of the issuing entity's home country serves as the global benchmark settlement channel, supported by transitional arrangements and binding compliance obligations for local tax and fund traceability |

## 5. 如何采用本范式 / How to Adopt This Paradigm

任何企业、团队或个人，可按照以下步骤采用 QNLOO-SPL 范式：

Any enterprise, team, or individual may adopt the QNLOO-SPL paradigm by following these steps:

1. 下载模板 Clone or download the `template/` directory

2. 替换资产持有主体信息 Replace the asset holding entity information

3. 在项目 README 中标注 Label your project as "本项目采用 QNLOO-SPL 主权公开许可范式 / Licensed under the QNLOO-SPL Sovereign Public License Paradigm"

4. 核心法律条款禁止私自删减修改，仅可替换资产持有主体、本地化营收阈值等基础参数 Core legal clauses must not be altered or removed; only basic parameters such as the asset holding entity and localized revenue thresholds may be replaced
